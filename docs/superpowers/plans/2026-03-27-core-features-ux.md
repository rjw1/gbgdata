# Core Features & Map UX Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement a map view, an "open pubs only" toggle, regional list views, and an About page.

**Architecture:**
1. **Map View**: Use a map library (e.g., Leaflet via `leptos-leaflet` or direct JS interop) to display pub locations on the explorer page.
2. **Filtering**: Add a `closed=false` filter to the server functions and a toggle in the UI.
3. **Regional View**: Update the `RegionDashboard` (formerly `CountyDashboard`) to show a full list of pubs in the region, in addition to the town/outcode drill-down.
4. **About Page**: Create a new route and component for project info.

**Tech Stack:** Rust, Leptos, Leaflet (or similar), CSS/SCSS.
